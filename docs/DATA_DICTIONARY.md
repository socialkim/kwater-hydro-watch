# 📚 데이터 사전 (Data Dictionary) — HRFCO 수문 API

> 한강홍수통제소(HRFCO) Open API의 댐/수위/강수/보 데이터 필드 정의서입니다.
> 모든 필드·URL은 2026-06-15 국내 PC에서 실호출로 **검증 완료**한 사실 기반입니다(추측 아님).

---

## 0. 공통 사항 (Common)

| 항목 | 값 |
|------|-----|
| **Base URL** | `http://api.hrfco.go.kr` (`https`도 가능) |
| **응답 형식** | JSON. 실제 데이터는 `content` 배열에 담김: `{ "links":[], "content":[ {...}, ... ] }` |
| **인증** | URL 경로에 인증키 포함 (`/{KEY}/...`) |
| **CORS** | 개방됨 (`Access-Control-Allow-Origin: *`) → 브라우저 직접 호출 가능 |
| **정렬 주의** | 시계열은 최신이 배열 앞쪽(내림차순)일 수 있음 → 사용 전 `ymdhm` 기준 정렬 권장 |

### URL 패턴

| 용도 | 패턴 |
|------|------|
| 관측소 목록 (info) | `/{KEY}/{type}/info.json` |
| 시계열 (data) | `/{KEY}/{type}/list/{TIME}/{obscode}/{sdt}/{edt}.json` |

- `{type}` = `dam` \| `waterlevel` \| `rainfall` \| `bo`
- `{TIME}` = `10M`(10분) \| `1H`(1시간) \| `1D`(1일)
- `{sdt}`, `{edt}` = `YYYYMMDDHHmm` (예: `202606150000`)

### 공통 정보 필드 (info)

| 필드 | 한글명 | 단위/형식 | 설명 |
|------|--------|-----------|------|
| `obsnm` | 관측소명 | 문자열 | 댐/관측소 이름 |
| `agcnm` | 관리기관 | 문자열 | 관할 기관명 |
| `addr` | 주소 | 문자열 | 소재지 주소 |
| `etcaddr` | 상세주소 | 문자열 | 보조 주소 (보 제외) |
| `lon` | 경도 | DMS 문자열 | `"127-59-44"` 형태 (도-분-초) |
| `lat` | 위도 | DMS 문자열 | `"37-56-52"` 형태 (도-분-초) |

## 1. 댐 (dam)

### 1-1. 정보 필드 (`/{KEY}/dam/info.json`)

| 필드 | 한글명 | 단위 | 설명 |
|------|--------|------|------|
| `dmobscd` | 댐 관측소 코드 | — | 시계열 조회용 고유 코드 (예: `1003110` 충주댐) |
| `obsnm` | 댐 이름 | — | |
| `agcnm` | 관리기관 | — | |
| `addr` / `etcaddr` | 주소 / 상세주소 | — | |
| `lon` / `lat` | 경도 / 위도 | DMS 문자열 | |
| `pfh` | 계획홍수위 | EL.m | 댐 설계상 최고 허용 수위 |
| `fldlmtwl` | 홍수기제한수위 | EL.m | 홍수기 운영 상한 수위 (공백일 수 있음) |

### 1-2. 시계열 데이터 필드 (`/{KEY}/dam/list/{TIME}/{code}/{sdt}/{edt}.json`)

| 필드 | 한글명 | 단위 | 설명 |
|------|--------|------|------|
| `ymdhm` | 관측 시각 | `YYYYMMDDHHmm` | |
| `swl` | 저수위 (현재 수위) | EL.m | 위험도 판정의 기준값 |
| `inf` | 유입량 | ㎥/s | 댐으로 들어오는 물의 양 |
| `sfw` | 저수량 | 백만㎥ | 현재 댐에 저장된 물의 양 |
| `ecpc` | 공용량(홍수조절 가용용량) | 백만㎥ | 추가로 받아낼 수 있는 여유 용량 |
| `tototf` | 총방류량 | ㎥/s | 댐에서 내보내는 물의 총량 |

> ✅ 저수량 필드는 **`sfw`** 입니다(2026-06-15 충주댐 실호출로 확인). 웹앱·MCP 모두 `sfw`를 사용합니다.

### 1-3. 위험도 판정 로직 (Dam)

현재 저수위 `swl`을 제한수위·계획홍수위와 비교합니다.

| 단계 | 조건 | 의미 |
|------|------|------|
| 🟢 정상 | `swl` < `fldlmtwl` | 평상 운영 |
| 🟡 주의 | `swl` ≥ `fldlmtwl` | 홍수기제한수위 초과 |
| 🔴 위험 | `swl` ≥ `pfh` | 계획홍수위 도달/초과 |

> `fldlmtwl`이 공백이면 주의 단계 판정에서 제외합니다.

### 2-2. 시계열 데이터 필드

| 필드 | 한글명 | 단위 | 설명 |
|------|--------|------|------|
| `ymdhm` | 관측 시각 | `YYYYMMDDHHmm` | |
| `wl` | 수위 | m | 현재 하천 수위 |
| `fw` | 유량 | ㎥/s | 하천 흐름량 |

### 2-3. 홍수 단계 판정 로직 (Water Level)

현재 수위 `wl`을 4단계 임계값과 비교합니다 (낮음 → 높음).

| 단계 | 조건 | 색상 |
|------|------|------|
| 🟢 정상 | `wl` < `attwl` | 초록 |
| 🟡 주의 | `attwl` ≤ `wl` < `wrnwl` | 노랑 |
| 🟠 경계 | `wrnwl` ≤ `wl` < `almwl` | 주황 |
| 🔴 경보 | `almwl` ≤ `wl` < `srswl` | 빨강 |
| ⚫ 심각 | `wl` ≥ `srswl` | 검정/심각 |

> 공백인 임계값은 해당 단계 판정에서 **제외**합니다 (예: `srswl`이 공백이면 심각 단계 미적용).

## 5. 좌표 변환 — DMS → 십진수 (Decimal Degrees)

`lon`/`lat`은 **도-분-초(DMS) 문자열** (`"127-59-44"`)로 제공됩니다. 지도(Leaflet) 표시를 위해 십진수로 변환합니다.

### 공식
```
십진수 = 도(D) + 분(M)/60 + 초(S)/3600
예) "127-59-44" → 127 + 59/60 + 44/3600 = 127.9956
```

### 변환 함수 (JavaScript)
```js
function dms2dec(s){
  const [d, m, sec] = s.trim().split('-').map(Number);
  return d + m/60 + (sec || 0)/3600;
}
```
> 🔎 **주의:** 문자열 끝 공백이 있을 수 있으므로 `trim()` 필수. 초(`sec`)가 없으면 0으로 처리.
